## 1. SAKANA FUGU (sakana.ai/fugu) — Our Direct Competitor

### Design
- **Theme**: Light (white #FFFFFF, text #111827)
- **Font**: Poppins + Noto Sans JP (Japanese support)
- **H1**: 71.68px, weight 500, color #111827
- **Cards**: No borders, no radius, padding 26px 24px — very flat, editorial style
- **Nav**: Transparent, sticky, with language selector (JP/EN toggle)
- **Hero**: Video animation of multi-agent system, headline "One Model to Command Them All"
- **CTAs**: "Start Using Sakana Fugu" (primary), "See the technology" (secondary)

### Key UX Patterns We Should Learn From
1. **"One Model to Command Them All"** — positions orchestration as a single model, not a framework. Exactly what we're doing.
2. **Availability badges**: Shows logos of platforms where Fugu is available (OpenRouter, Vercel, opencode, Creao, Merge). We should show "Available on OpenRouter, Ollama, ai/ml"
3. **Numbered benefits (01, 02, 03)**: Clean editorial numbering for feature cards
4. **Research papers as social proof**: Shows TRINITY and Conductor paper covers with "PAPER" links. We should link to our research docs.
5. **Two model tiers shown side by side**: "Fugu" (balanced) and "Fugu Ultra" (performance). Like our Quick vs Fusion modes.
6. **Benchmark table**: Full comparison table with bold for highest, underline for second. We should do this.
7. **"Plug collective intelligence directly into your workflows"** — great tagline pattern
8. **"Not yet available in EU/EEA"** — compliance disclaimer shown upfront. Honest.
9. **Quantitative results section**: Chart image + detailed table with footnotes. Very academic, builds credibility.
10. **Users' Voices section**: Testimonials from real users. Social proof.

### What Fugu Does Better Than Us (That We Must Match)
- Academic credibility (ICLR papers, technical report links)
- Benchmark table with full methodology and footnotes
- Platform availability badges
- Clean editorial layout without card borders

### What We Can Do Better
- Show orchestration happening in real-time (Fugu doesn't have a playground)
- Open source (Fugu is proprietary)
- Free tier with Ollama (Fugu is paid only)
- Warm design (Fugu is plain white, no personality)

## 2. HERMES AGENT (hermes-agent.nousresearch.com) — Our Own Platform

### Design
- **Theme**: Pure blue (#0000F2) background — extremely bold, unique
- **Text**: #F5F5F5 (near-white on blue)
- **Font**: System sans-serif
- **H1**: 71.59px, weight 300 (light), letter-spacing 2.15px (wide, airy)
- **Nav**: Transparent, minimal (NOUS, DOCS, HERMES AGENT, Discord, GitHub, PORTAL, INSTALL)

### Key UX Patterns
1. **Bold color choice**: Pure blue #0000F2 is unlike any other AI site. Memorable.
2. **Light weight H1**: 300 weight at 71px creates an airy, confident feel (not shouting)
3. **Wide letter-spacing on H1**: 2.15px creates a premium, editorial feel
4. **Install command with tabs**: macOS/Linux vs Windows tabs, copy button. Immediate "get started" without scrolling.
5. **Platform downloads**: Three cards (macOS, Windows, Linux) with clear OS labels
6. **Feature preview section**: Article cards with 2-word headings ("LIVES EVERYWHERE", "PERSISTENT MEMORY") — very punchy
7. **Video element**: Shows the product in action
8. **Minimal footer**: Just "NOUS PORTAL" + "VIEW ALL OUR PLANS"

### Lessons for Timuclaude
- A bold, unexpected color can be more memorable than a "safe" palette
- Light-weight headings at large sizes feel premium
- Install command in the hero = immediate action (no scrolling to find how to start)
- 2-word feature headings are punchy and memorable

## 3. GOOGLE AI STUDIO (aistudio.google.com) — The Reference Standard

### Design
- **Theme**: Dark (#121317), white text
- **Font**: Inter, sans-serif
- **H1**: 64px, weight 450 (variable font)
- **Hero**: Video background, carousel slider
- **Nav**: Google AI Studio logo, Pricing, Documentation, Case studies, Get started

### Key UX Patterns
1. **Video hero**: Auto-playing video showing the product in action
2. **Model carousel**: "Experiment with the latest models" — horizontal scrolling cards (Gemini 3, Nano Banana, Veo, TTS, Live, Lyria)
3. **4-step getting started**: API key → SDKs → Interactions API → Tools. Each with icon + description + "Start" link
4. **"Projects ready to Remix"**: Template gallery with categories (Featured, Personal websites, 2D & 3D games, Interactive art)
5. **"Bring your ideas to life"**: Text input with suggestion chips ("A conversational language learning app", "An app to remove backgrounds")
6. **Case studies section**: "How developers build with Gemini API" — 3 case study links
7. **Comprehensive footer**: Models, Product, Capabilities, Information — 4 columns

### Lessons for Timuclaude
- Model carousel is a great way to show available models (we can show our 6 models)
- "Projects ready to Remix" = "Example prompts ready to try" in our playground
- Getting started steps (API key → SDK → Tools) = our Quickstart flow
- Suggestion chips under the prompt input = great UX for first-time users

## 4. MINIMAX (minimax.io) — Multi-Product Showcase

### Design
- **Theme**: Light (white #FFFFFF, text #18181B)
- **Font**: DM Sans + Helvetica Neue
- **H1**: Small (16px in carousel, product names)
- **Hero**: Carousel/slider with product cards (M3, Code, Token Plan)
- **Nav**: Mega menu with product categories (Models, Products, API, Company)

### Key UX Patterns
1. **Carousel hero**: Multiple products featured in a slider with prev/next arrows
2. **Mega menu navigation**: Grouped products (MiniMax M3, M2.7, M2.5, Hailuo, Speech, Music)
3. **"Flagship Models" section**: 3 large cards showcasing top models with images
4. **"Latest Research" section**: Blog post cards with date, category, and "Read More"
5. **Cookie consent**: Full cookie consent banner with Reject/Customize/Accept
6. **Footer**: 4 columns (Model, Product, API, Company) with all sub-links

### Lessons for Timuclaude
- Mega menu works when you have many products (we have 6 models, could group them)
- Research blog cards with dates build credibility over time
- Footer with categorized links is standard and expected

## 5. KILO AI (kilo.ai) — Open-Source Developer Tool (25.4K GitHub Stars)

### Design
- **Theme**: Dark (#0A0A0A), white text
- **Font**: JetBrains Mono (entire site uses monospace!)
- **H1**: 72px, weight 700, letter-spacing -1.8px (tight, bold)
- **Hero**: GitHub stars badge (25.4k), headline, platform selector, install command

### Key UX Patterns — VERY RELEVANT TO US
1. **GitHub stars in hero**: "25.4k" stars badge right at the top. Social proof.
2. **Platform selector**: Radio buttons (VS Code / JetBrains / CLI / Cloud) with install command that changes. We should do this for Ollama / OpenRouter / ai/ml.
3. **"Code with AI without breaking the bank"**: Cost-focused headline. We should use "Beat frontier models without the frontier price."
4. **"500+ models, zero markup"**: Transparent pricing in the subtext. We should say "5+ models, 28x cheaper than Fable 5."
5. **"No black boxes. Inspect everything."**: Open-source positioning. We should use this exact framing.
6. **Trusted by logos**: Meta, Amazon, Airbnb, PayPal, Square, Red Hat — enterprise social proof
7. **Metrics bar**: "#1 Open Source Product of the Month", "3M+ Kilo Coders", "40T+ tokens processed"
8. **Feature cards with "AGENT COMMAND CENTER", "ALIGNED PRICING", "OPEN SOURCE"**: Labeled categories with bullet lists and "Start using" / "View on GitHub" links
9. **"No credit card required"**: Friction removal
10. **"Bring your own keys"**: Same as our Ollama/OpenRouter approach
11. **Entire site in JetBrains Mono**: Bold choice that says "this is for developers." Unique and memorable.

### Lessons for Timuclaude
- GitHub stars in the hero = immediate credibility
- Platform/backend selector with dynamic install command = great UX
- "No markup", "No black boxes", "Bring your own keys" = transparent pricing language
- Metrics bar (#1 product, X users, Y tokens) = quantified social proof
- "No credit card required" = removes biggest signup friction
- Monospace font for entire site = bold developer signal (but we'll use it only for code blocks, keep Inter for body)

- Full benchmark table with footnotes
- Platform availability badges (OpenRouter, Vercel, etc.)

- Install command in hero = immediate action
- 2-word punchy feature headings

- Model carousel showing available models
- Template/example gallery with categories
- 4-step getting started flow
- Suggestion chips under prompt input

### From MiniMax (multi-product)
- Research blog cards with dates
- Mega menu for many products
- Cookie consent banner

### From Kilo AI (open-source dev tool — MOST relevant)
- GitHub stars badge in hero
- Backend/platform selector with dynamic install command
- "No markup", "No black boxes", "Bring your own keys" language
- Metrics bar (#1 product, X users, Y tokens)
- "No credit card required" friction removal
